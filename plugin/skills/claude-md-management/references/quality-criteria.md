# Quality Criteria Details

## Commands & Workflows (20 points)

- **20**: All essential commands documented (build, test, lint, format, deploy), with environment prerequisites
- **15**: Most commands present, minor gaps
- **10**: Only basic commands, missing important workflows
- **5**: Minimal or outdated commands
- **0**: No commands documented

## Architecture Clarity (20 points)

- **20**: Clear directory structure, key patterns explained, data flow documented, dependency relationships clear
- **15**: Good structure overview, some patterns missing
- **10**: Basic directory listing without explanation
- **5**: Vague references to architecture
- **0**: No architecture information

## Non-Obvious Patterns (15 points)

- **15**: Documents gotchas, workarounds, and surprising behaviors. Explains WHY, not just WHAT.
- **10**: Some non-obvious patterns documented
- **5**: Only obvious information
- **0**: No gotchas or workarounds mentioned

## Conciseness (15 points)

- **15**: Every line is necessary and informative. No repetition. Bullet points and tables over prose.
- **10**: Mostly concise with some fluff
- **5**: Verbose, repetitive, or padded
- **0**: Wall of text with low information density

## Currency (15 points)

- **15**: All references match current codebase. No mentions of removed features, renamed files, or deprecated patterns.
- **10**: Mostly current, minor stale references
- **5**: Multiple outdated references
- **0**: Significantly out of date

## Actionability (15 points)

- **15**: Instructions can be followed verbatim. Commands are copy-pasteable. Prerequisites are explicit.
- **10**: Mostly actionable, some implicit knowledge assumed
- **5**: Vague guidance like "set up the database"
- **0**: Abstract descriptions with no concrete steps
