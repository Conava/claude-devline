# Aesthetics Guide — Distinctive Frontend Design

## Philosophy

Claude is capable of extraordinary creative work. Don't hold back — show what can truly be created when thinking outside the box and committing fully to a distinctive vision.

Every interface should feel like it was designed by a human with strong opinions, not generated by a committee. The goal is production-grade code that people remember.

## The Design Process

### 1. Choose an Aesthetic Direction

Before writing any code, commit to a specific aesthetic. Some directions to consider:

| Direction | Character | Key Elements |
|-----------|-----------|-------------|
| Brutally Minimal | Stark, intentional emptiness | Massive whitespace, one accent color, dramatic type scale |
| Maximalist Chaos | Overwhelming density, visual feast | Overlapping layers, clashing colors, mixed media |
| Retro-Futuristic | Vintage sci-fi meets modern tech | CRT scan lines, neon on dark, monospace type, glitch effects |
| Organic/Natural | Warm, flowing, handmade feel | Soft curves, earth tones, hand-drawn elements, paper textures |
| Luxury/Refined | Premium, understated elegance | Serif typography, muted metallics, generous spacing, subtle motion |
| Playful/Toy-like | Fun, bouncy, childlike wonder | Rounded shapes, bright primaries, springy animations, oversized elements |
| Editorial/Magazine | Print-inspired digital layout | Column grids, pull quotes, dramatic image crops, sophisticated type |
| Brutalist/Raw | Ugly-beautiful, anti-design | Default system styling, raw borders, exposed grid, confrontational |
| Art Deco/Geometric | Glamorous precision | Gold accents, geometric patterns, symmetry, ornamental lines |
| Soft/Pastel | Gentle, approachable calm | Light pastels, rounded corners, soft shadows, dreamy gradients |
| Industrial/Utilitarian | Functional, no-nonsense | Monospace, data-dense, minimal decoration, dark backgrounds |

**Mix and subvert these.** The best designs often combine elements from multiple directions in unexpected ways.

### 2. Typography Selection

Typography is the single biggest factor in making a design feel distinctive vs. generic.

**How to choose:**
- Match the typeface personality to the aesthetic direction
- A luxury interface needs an elegant serif, not a geometric sans
- A brutalist interface needs raw, aggressive type, not a friendly rounded font
- A retro-futuristic interface needs tech-inspired monospace or angular display fonts

**Pairing strategy:**
- One distinctive display/heading font (the personality)
- One refined body font (the workhorse)
- Optional: one mono font for code/data

**Sources for interesting fonts:**
- Google Fonts (free): look beyond the first page
- Variable fonts for fine-grained weight control
- System fonts only when the design intentionally calls for raw/brutalist aesthetic

### 3. Color Strategy

**Building a palette:**
1. Start with the mood — warm vs. cool, light vs. dark
2. Choose a dominant color (60% of the design)
3. Choose a sharp accent (10% — this is what people remember)
4. Fill the rest with neutrals (30%)
5. Test contrast ratios for accessibility

**Color relationships:**
- Complementary (opposite on color wheel): high energy, vibrant
- Analogous (neighbors on color wheel): harmonious, calming
- Monochromatic (shades of one hue): sophisticated, cohesive
- Split-complementary: vibrant but more balanced than pure complementary

**Dark themes:** Not just "invert the colors." Dark themes need:
- Slightly desaturated colors (pure bright colors hurt on dark backgrounds)
- Reduced contrast from pure white text (use off-white: #E0E0E0 - #F0F0F0)
- Elevated surfaces with subtle lightening, not borders
- Colored accents that pop against dark backgrounds

### 4. Motion Design

**High-impact moments to animate:**
- Page load: staggered reveals of content sections
- Route transitions: meaningful directional movement
- Data changes: smooth number counting, list reordering
- Hover states: unexpected, delightful micro-reactions
- Scroll events: parallax, reveal-on-scroll, sticky transformations

**Animation principles:**
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (quick start, gentle end) for most transitions
- Duration: 150-300ms for micro-interactions, 400-800ms for page transitions
- Stagger: 50-100ms delay between sequential elements
- Spring physics for playful/natural feel (Motion library, Framer Motion)

**CSS-only techniques:**
```css
/* Staggered reveal */
.item { animation: fadeUp 0.6s ease-out both; }
.item:nth-child(1) { animation-delay: 0.0s; }
.item:nth-child(2) { animation-delay: 0.1s; }
.item:nth-child(3) { animation-delay: 0.2s; }

@keyframes fadeUp {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```

### 5. Texture & Depth

Flat design is a starting point, not a destination. Add depth with:

- **Noise/grain overlay:** `background-image: url('data:image/svg+xml,...')` with SVG noise
- **Gradient meshes:** Multiple radial gradients layered with blend modes
- **Glassmorphism:** `backdrop-filter: blur()` with semi-transparent backgrounds
- **Shadows:** Multi-layered shadows for realistic depth, or dramatic single shadows for drama
- **Borders:** Subtle gradients, dashed patterns, or thick colored borders as design elements

### 6. Layout Composition

Break free from the standard top-to-bottom flow:

- **Overlapping elements:** Negative margins, absolute positioning for intentional overlap
- **Asymmetric grids:** 2/3 + 1/3 splits, staggered card heights, masonry layouts
- **Diagonal flow:** `transform: skewY()` on section backgrounds, angled dividers
- **Full-bleed elements:** Break out of the container for impact
- **Bento grid:** Mixed-size cards in a tight grid (popularized by Apple)

## Anti-Pattern Gallery

### Generic "AI Slop" Characteristics

These patterns instantly signal "AI-generated" and should be avoided:

1. **The Purple Gradient Hero** — White background, purple-to-blue gradient accent, Inter font, centered text, rounded cards with subtle shadows
2. **The SaaS Template** — Navbar with logo left / links right, hero with big headline + subtext + two buttons, three feature cards, testimonial section, CTA, footer
3. **The Dashboard Clone** — Sidebar with icons, top bar with search + avatar, card grid with charts, all in gray/blue/white
4. **The Landing Page Kit** — Alternating left-right sections with image + text, generic stock-photo-style illustrations

**Why these fail:** They're not bad designs — they're just forgettable. They signal "I didn't think about this, I just used a template."

### How to Break Free

- Start with the CONTENT, not the layout
- Ask "what would a human designer obsess over here?"
- Find one element to make extraordinary — a single stunning animation, an unusual type treatment, a bold color choice
- Let that element inform the rest of the design
- Remove elements until it hurts, then add one thing back

## Implementation Quality

### CSS Variables
```css
:root {
  /* Don't just define colors — define your design system */
  --space-unit: 8px;
  --space-xs: calc(var(--space-unit) * 0.5);
  --space-sm: var(--space-unit);
  --space-md: calc(var(--space-unit) * 2);
  --space-lg: calc(var(--space-unit) * 4);
  --space-xl: calc(var(--space-unit) * 8);

  --font-display: 'Your Distinctive Choice', serif;
  --font-body: 'Your Workhorse Choice', sans-serif;

  --color-dominant: #...;
  --color-accent: #...;
  --color-surface: #...;
  --color-text: #...;
  --color-text-muted: #...;
}
```

### Production Quality Checklist
- [ ] Works at all viewport sizes (320px to 2560px)
- [ ] Keyboard navigable
- [ ] Color contrast passes WCAG AA
- [ ] Animations respect prefers-reduced-motion
- [ ] No layout shifts on load
- [ ] Images have alt text
- [ ] Interactive elements have visible focus states
- [ ] Design is cohesive — every element serves the aesthetic direction
